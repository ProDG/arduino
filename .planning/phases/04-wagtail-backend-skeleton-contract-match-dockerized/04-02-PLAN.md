    ```python
    from storages.backends.s3 import S3Storage


    class MediaS3Storage(S3Storage):
        """Default storage for Wagtail Image originals + Document originals + auto-generated
        renditions.

        querystring_auth=False emits unsigned URLs for ALL files. Bucket policy gates
        public access per-prefix:
          - originals/  : no anonymous policy → 403 to public (admin uploads via authed S3 calls)
          - images/     : public-read policy  → 200 to public (renditions, cacheable, prerender-safe)
          - documents/  : no anonymous policy → 403 to public
        """

        location = ""
        file_overwrite = False
        querystring_auth = False
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public internet → Traefik /media/* → MinIO | Anonymous reads allowed ONLY for `images/` prefix |
| Wagtail container → MinIO :9000 (Docker DNS) | S3 API auth via `MINIO_ROOT_USER`/`MINIO_ROOT_PASSWORD` |
| Developer host → MinIO :9000/:9001 (dev only) | Convenience for `mc` ad-hoc + console UI |
| Wagtail admin upload → MinIO PUT | Authenticated S3 call from server-side; admin session gates access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-10 | Information Disclosure | Public read of original uploads (PII, unpublished imagery) | mitigate | `mc anonymous set download` applied ONLY to `arduino-media/images/`; `originals/` and `documents/` left at default (no anonymous policy → 403); acceptance test: `curl http://arduino.localhost/media/arduino-media/originals/...` returns 403 |
| T-04-11 | Information Disclosure | MinIO root credentials in `.env` | mitigate | `MINIO_ROOT_USER`/`MINIO_ROOT_PASSWORD` env-driven; gitleaks pre-commit blocks committed `.env`; rotation possible by stack restart |
| T-04-12 | Tampering | Bucket-policy drift between dev and prod | mitigate | `mc anonymous set download local/arduino-media/images` runs in the same `mc` sidecar that ships in P5 prod compose; idempotent → re-runnable safely; acceptance test grep ensures only `images` prefix gets the policy |
| T-04-13 | Spoofing | MinIO uses virtual-host-style addressing | mitigate | `AWS_S3_ADDRESSING_STYLE='path'` set explicitly (Pitfall 2); failure mode is loud DNS error so regressions are immediately visible |
| T-04-14 | Denial of Service | Rendition flood (CPU on first generation, MinIO disk fill) | accept | P4 has no real content; P5 deploys quarterly cleanup cron `wagtail_update_image_renditions --purge-only` (DEPLOY-06); not a P4 concern |
| T-04-15 | Information Disclosure | MinIO console :9001 exposed to host in dev | accept | Dev convenience only; never exposed in prod (P5 omits `compose.dev.yml`); acceptance test confirms publish only in dev overlay |
| T-04-16 | Elevation of Privilege | S3 access keys reused as MinIO root | mitigate | Dev uses root creds for simplicity (acceptable for ephemeral dev MinIO); P5 issues scoped service-account keys for prod via `mc admin user add`; documented in deferred-to-P5 note |
</threat_model>
