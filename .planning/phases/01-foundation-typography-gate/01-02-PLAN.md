<objective>
Build the font pipeline: download SIL-OFL source TTFs for Source Serif 4, Inter, and JetBrains Mono; subset each to latin + latin-ext + cyrillic + cyrillic-ext (preserving variable axes); produce six variable woff2 files; generate Fontaine fallback metric overrides as a committed JSON sidecar that PLAN 03 will inline into `_typography.scss`.

Purpose: Cyrillic-Ext coverage is non-negotiable (`ґ` U+0490/U+0491 lives there). Real italics (not synthesized) require a separate italic woff2 per family. Fontaine metrics eliminate CLS on font swap. This plan is the supply-chain-controlled font pipeline.
Output: Six woff2 files in `public/fonts/`, a re-runnable subsetting script, license files, and a Fontaine metrics JSON sidecar.
</objective>

    Create the font supply chain. The script must be deterministic and re-runnable.

    1. **Download source TTFs** into `fonts-source/` (gitignored per PLAN 01 .gitignore):

       - Inter — pull `Inter[slnt,wght].ttf` and `Inter-Italic[wght].ttf` (or equivalent variable italic file from the latest Inter release at https://github.com/rsms/inter/releases). If Inter ships a single `slnt` variable file rather than separate italic, extract italic instances per the latest release notes — the goal is a separate italic woff2 with real italic glyphs, NOT synthesized via `font-style: oblique`.
       - JetBrains Mono — pull `JetBrainsMono[wght].ttf` and `JetBrainsMono-Italic[wght].ttf` from https://github.com/JetBrains/JetBrainsMono/releases.

       Use `curl -L -o` with the version tag from `sources.json`. Compute SHA-256 of each downloaded TTF and write to `public/fonts/CHECKSUMS.txt` for supply-chain traceability. (Do NOT verify against a hardcoded value — record what was downloaded so future bumps can compare.)

       Copy each font's `OFL.txt` to `public/fonts/LICENSES/{family}-OFL.txt`.

    2. **Create `scripts/fonts/sources.json`** — pinned version manifest:
       ```json
       {
         "source-serif-4": { "version": "<resolved-tag>", "url-roman": "...", "url-italic": "..." },
         "inter": { "version": "<resolved-tag>", "url-roman": "...", "url-italic": "..." },
         "jetbrains-mono": { "version": "<resolved-tag>", "url-roman": "...", "url-italic": "..." }
       }
       ```
       Resolve `<resolved-tag>` by checking the latest stable release at script-run time and pinning it.

    3. **Create `scripts/fonts/subset.mjs`** — Node ESM script wrapping `pyftsubset`. It MUST:
       - Verify `pyftsubset` is on PATH; if not, instruct user to `pip install fonttools[woff]` or `uv tool install fonttools` and exit 1.
       - For each of the six (family × style) inputs, invoke `pyftsubset` with these flags (D-10, D-11, D-12):
         ```
         pyftsubset <input.ttf> \
           --output-file=<public/fonts/{name}.woff2> \
           --flavor=woff2 \
           --layout-features=* \
           --unicodes=U+0000-007F,U+0080-024F,U+0400-04FF,U+0500-052F,U+2DE0-2DFF,U+1E00-1EFF,U+2000-206F,U+2070-209F,U+20A0-20CF,U+2100-214F,U+2C60-2C7F,U+A640-A69F,U+02BC \
           --no-hinting \
           --desubroutinize \
           --drop-tables+=DSIG \
           --name-IDs=* \
           --notdef-outline
         ```
         The `--unicodes` set covers Latin Basic + Latin-1 Supplement + Latin Extended-A/B + **Cyrillic (U+0400-04FF) + Cyrillic Supplement (U+0500-052F) + Cyrillic Extended-A (U+2DE0-2DFF) + Cyrillic Extended-B (U+A640-A69F)** + Latin Extended Additional + General Punctuation + Superscripts + Currency + Letterlike + Latin Extended-C + the typographic apostrophe `ʼ` (U+02BC). Also append `U+2DE0-2DFF` to ensure Cyrillic Extended-A is included. **Verify ґ (U+0491) and Ґ (U+0490) are inside the included ranges** — they are at U+0490/U+0491 inside Cyrillic Supplement boundary; the script MUST log "ґ U+0491 included" for each output as a sanity print.
       - Preserve variable axes — `pyftsubset` keeps them by default unless `--no-recalc-bounds` strips them; do NOT pass any axis-stripping flag.
       - After each subset, programmatically re-open the woff2 with fonttools (via `python -c` or a sibling Python helper) and assert: (a) `cmap` contains glyphs for U+0490, U+0491, U+0457, U+0454, U+0456, U+02BC; (b) `fvar` table exists (variable font); (c) for Source Serif 4, both `wght` and `opsz` axes present; for Inter and JetBrains Mono, `wght` axis present. Fail loudly with the missing glyph/axis if any check fails.
       - Output filenames per D-09:
         - `source-serif-4-roman.woff2`, `source-serif-4-italic.woff2`
         - `inter-roman.woff2`, `inter-italic.woff2`
         - `jetbrains-mono-roman.woff2`, `jetbrains-mono-italic.woff2`
       - Print final size table (each file's size in KB).

    4. **`scripts/fonts/README.md`** — short doc:
       - Why subset (Cyrillic-Ext required for ґ, single subset per file per D-11).
       - How to run: `pyftsubset` prerequisite install, `node scripts/fonts/subset.mjs`.
       - When to re-run: only on a font release bump; outputs are committed.
       - Link to D-08, D-09, D-10, D-11 in CONTEXT.md.

    5. **Add npm scripts** to `package.json`:
       - `"fonts:subset": "node scripts/fonts/subset.mjs"`
       - `"fonts:metrics": "node scripts/fonts/fontaine-metrics.mjs"` (Task 2 below).

    6. **Run the script.** Commit the six resulting woff2 files. Reject any output > 250KB per file (Cyrillic-Ext + Latin-Ext on a variable font typically lands 80–180KB; if a file is bigger, investigate before committing).

    Reference D-08 (pyftsubset wrapped in subset.mjs), D-09 (six woff2 files), D-10 (latin + latin-ext + cyrillic + cyrillic-ext), D-11 (single subset, no unicode-range split).
